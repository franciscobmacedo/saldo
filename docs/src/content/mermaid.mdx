## Data Flow Process

```mermaid
sequenceDiagram
    participant User
    participant Validator
    participant SituationUtils
    participant TaxRetentionTable
    participant Calculator
    
    User->>Validator: simulateDependentWorker(options)
    Validator->>Validator: validateNumberOfHolders()
    Validator->>Validator: validateMarriedAndNumberOfHolders()
    Validator->>Validator: validateDependents()
    
    Validator->>SituationUtils: getSituation(married, disabled, holders, dependents)
    SituationUtils->>SituationUtils: Determine tax situation code
    SituationUtils-->>Validator: Return situation
    
    Validator->>TaxRetentionTable: load(dates, location, situationCode)
    TaxRetentionTable->>TaxRetentionTable: Load appropriate tax table
    TaxRetentionTable-->>Validator: Return tax table
    
    Validator->>Calculator: calculate(income, allowances, deductions)
    Calculator->>TaxRetentionTable: find_bracket(taxableIncome)
    TaxRetentionTable-->>Calculator: Return tax bracket
    Calculator->>Calculator: Apply tax calculations
    Calculate->>Calculator: Apply social security
    Calculator-->>User: Return DependentWorkerResult
```

## Tax Calculation Flow

```mermaid
graph TD
    Income[Base Income] --> LunchCalc{Lunch Allowance?}
    LunchCalc -->|Yes| AddLunch[Add Taxable Portion]
    LunchCalc -->|No| TaxableIncome[Taxable Income]
    AddLunch --> TaxableIncome
    
    TaxableIncome --> Twelfths{Holiday Allowances?}
    Twelfths -->|Yes| AddTwelfths[Add Twelfths Income]
    Twelfths -->|No| RetentionIncome[Retention Income]
    AddTwelfths --> RetentionIncome
    
    RetentionIncome --> TaxBracket[Find Tax Bracket]
    TaxBracket --> Deductions{Extra Deductions?}
    
    Deductions -->|Partner Disabled| PartnerDed[Partner Deduction]
    Deductions -->|Disabled Dependents| DepDed[Dependent Deduction]
    Deductions -->|None| CalcTax[Calculate Tax]
    
    PartnerDed --> CalcTax
    DepDed --> CalcTax
    
    CalcTax --> SocialSec[Calculate Social Security]
    SocialSec --> NetSalary[Calculate Net Salary]
    NetSalary --> YearlyCalc[Calculate Yearly Amounts]
    YearlyCalc --> Result[Return Result]
```

## Regional Data Structure

```mermaid
graph TD
    Portugal[Portugal] --> Continente[Continente]
    Portugal --> Acores[Açores]
    Portugal --> Madeira[Madeira]
    
    Continente --> ContTables[Tax Tables 2023-2024]
    Acores --> AcoresTables[Regional Tax Tables]
    Madeira --> MadeiraTables[Regional Tax Tables]
    
    ContTables --> Situations[Tax Situations]
    AcoresTables --> Situations
    MadeiraTables --> Situations
    
    Situations --> SOLCAS2_Table[SOLCAS2]
    Situations --> SOLD_Table[SOLD]
    Situations --> CAS1_Table[CAS1]
    Situations --> Disability_Tables[+DEF Variants]
```

The architecture is designed for:
- **Modularity**: Clear separation of concerns
- **Extensibility**: Easy to add new regions or tax rules
- **Maintainability**: Well-defined interfaces and validation
- **Performance**: Efficient tax table loading and calculation
- **Type Safety**: Full TypeScript coverage for all operations
