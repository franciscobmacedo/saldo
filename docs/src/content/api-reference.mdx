# API Reference

Complete reference for all functions, types, and interfaces in Saldo TS.

## Main Function

### `simulateDependentWorker(options)`

The primary function for calculating Portuguese dependent worker taxes and net salary.

#### Parameters

```typescript
interface SimulateDependentWorkerOptions {
  income: number;
  married?: boolean;
  disabled?: boolean;
  partnerDisabled?: boolean;
  location?: LocationT;
  numberOfHolders?: number | null;
  numberOfDependents?: number | null;
  numberOfDependentsDisabled?: number | null;
  dateStart?: Date;
  dateEnd?: Date;
  socialSecurityTaxRate?: number;
  twelfths?: Twelfths;
  lunchAllowanceDailyValue?: number;
  lunchAllowanceMode?: "cupon" | "salary" | null;
  lunchAllowanceDaysCount?: number;
}
```

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `income` | `number` | **Required** | Monthly gross income in euros |
| `married` | `boolean` | `false` | Whether the person is married |
| `disabled` | `boolean` | `false` | Whether the person has a disability |
| `partnerDisabled` | `boolean` | `false` | Whether the spouse has a disability |
| `location` | `LocationT` | `'continente'` | Tax region: `'continente'`, `'acores'`, or `'madeira'` |
| `numberOfHolders` | `number \| null` | `null` | Number of income holders in marriage (1 or 2) |
| `numberOfDependents` | `number \| null` | `null` | Number of dependents |
| `numberOfDependentsDisabled` | `number \| null` | `null` | Number of dependents with disabilities |
| `dateStart` | `Date` | `new Date(2025, 0, 1)` | Calculation period start date |
| `dateEnd` | `Date` | `new Date(2025, 11, 31)` | Calculation period end date |
| `socialSecurityTaxRate` | `number` | `0.11` | Social security tax rate (11%) |
| `twelfths` | `Twelfths` | `TWO_MONTHS` | Holiday allowance distribution |
| `lunchAllowanceDailyValue` | `number` | `10.2` | Daily lunch allowance amount in euros |
| `lunchAllowanceMode` | `"cupon" \| "salary" \| null` | `"cupon"` | Type of lunch allowance |
| `lunchAllowanceDaysCount` | `number` | `22` | Working days per month for lunch allowance |

#### Return Value

```typescript
interface DependentWorkerResult {
  taxableIncome: number;
  grossIncome: number;
  tax: number;
  socialSecurity: number;
  socialSecurityTax: number;
  netSalary: number;
  yearlyNetSalary: number;
  yearlyGrossSalary: number;
  lunchAllowance: LunchAllowance;
}
```

| Property | Type | Description |
|----------|------|-------------|
| `taxableIncome` | `number` | Monthly income subject to tax |
| `grossIncome` | `number` | Total monthly gross income including allowances |
| `tax` | `number` | Monthly tax amount |
| `socialSecurity` | `number` | Monthly social security contribution |
| `socialSecurityTax` | `number` | Social security tax rate applied |
| `netSalary` | `number` | Monthly net salary after deductions |
| `yearlyNetSalary` | `number` | Annual net salary |
| `yearlyGrossSalary` | `number` | Annual gross salary |
| `lunchAllowance` | `LunchAllowance` | Lunch allowance breakdown |

#### Example

```typescript
import { simulateDependentWorker } from 'saldo-ts';

const result = simulateDependentWorker({
  income: 1500,
  married: true,
  numberOfHolders: 1,
  numberOfDependents: 2,
  location: 'continente',
  lunchAllowanceDailyValue: 8.5,
  lunchAllowanceMode: 'cupon',
  lunchAllowanceDaysCount: 22
});

console.log(`Net Salary: €${result.netSalary.toFixed(2)}`);
```

## Types and Enums

### `LocationT`

```typescript
type LocationT = "continente" | "acores" | "madeira";
```

Represents the three Portuguese tax regions:
- `continente`: Mainland Portugal
- `acores`: Azores region
- `madeira`: Madeira region

### `Twelfths`

```typescript
enum Twelfths {
  NONE = 0,        // No holiday allowances
  ONE_MONTH = 1,   // Either Christmas OR holiday allowance
  TWO_MONTHS = 2   // Both Christmas AND holiday allowance
}
```

Represents how holiday allowances (13th and 14th month) are distributed throughout the year.

### `LunchAllowance`

```typescript
interface LunchAllowance {
  dailyValue: number;
  mode: "cupon" | "salary" | null;
  daysCount: number;
  monthlyValue: number;
  taxableMonthlyValue: number;
  taxFreeMonthlyValue: number;
  yearlyValue: number;
}
```

| Property | Type | Description |
|----------|------|-------------|
| `dailyValue` | `number` | Daily lunch allowance amount |
| `mode` | `"cupon" \| "salary" \| null` | Type of lunch allowance |
| `daysCount` | `number` | Number of working days per month |
| `monthlyValue` | `number` | Total monthly lunch allowance |
| `taxableMonthlyValue` | `number` | Portion subject to tax |
| `taxFreeMonthlyValue` | `number` | Tax-free portion |
| `yearlyValue` | `number` | Annual lunch allowance (11 months) |

**Note**: This interface represents the structure of the `lunchAllowance` property in the result object. You don't create this object directly; instead, pass the lunch allowance parameters (`lunchAllowanceDailyValue`, `lunchAllowanceMode`, `lunchAllowanceDaysCount`) to the `simulateDependentWorker` function.

### `SituationCodesT`

```typescript
type SituationCodesT =
  | "SOLCAS2"      // Single or married with 2 holders
  | "SOLD"         // Single with dependents
  | "CAS1"         // Married, single holder
  | "SOLCAS2+DEF"  // Single or married 2 holders + disability
  | "SOLD+DEF"     // Single with dependents + disability
  | "CAS2D+DEF"    // Married 2 holders with dependents + disability
  | "CAS1+DEF";    // Married single holder + disability
```

Tax table codes used internally by the library to determine the appropriate tax bracket.

## Validation Functions

The library includes internal validation functions that are automatically called:

### `validateNumberOfHolders(numberOfHolders)`
Validates that the number of holders is valid (1 or 2 for married, null for single).

### `validateMarriedAndNumberOfHolders(married, numberOfHolders)`
Validates the relationship between marital status and number of holders.

### `validateDependents(numberOfDependents, numberOfDependentsDisabled)`
Validates that the number of disabled dependents doesn't exceed total dependents.

## Error Handling

The library throws descriptive errors for invalid configurations:

```typescript
// Invalid situation
throw new Error(
  `Could not determine situation for the given parameters: 
   married=${married}, disabled=${disabled}, 
   numberOfHolders=${numberOfHolders}, 
   numberOfDependents=${numberOfDependents}`
);

// Tax bracket not found
throw new Error(
  `Could not find tax bracket for taxable income: ${taxableIncome} 
   in table ${situation.code} for location ${location}`
);
```
